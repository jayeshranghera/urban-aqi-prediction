1. Overview

This project builds an hourly air-quality modelling pipeline and a Streamlit dashboard for four European cities. It combines **regulatory-grade monitoring data** where available (EEA) with **Open-Meteo** weather and, for Paris and Rome, **Open-Meteo CAMS-based air-quality fields**, so that all four cities share a consistent feature schema for machine learning.

The work is aimed at roles that blend **data science, environmental analytics, and policy communication**: predictions are paired with **EU-oriented concentration bands** (CSV-driven) and plain-language health guidance.

1.1 Motivation

Ambient PM2.5 and other pollutants remain a major public-health burden in Europe. The dashboard helps a non-specialist interpret **what concentrations imply for short-term behaviour**, while a technical reader can trace **sources, preprocessing, and model metrics** end to end.

1.2 Differentiators

вЂў Real ingestion patterns (EEA parquet + APIs), not a toy Kaggle-only exercise  
вЂў Explicit **data-provenance** table (Berlin/Madrid: EEA; Paris/Rome: Open-Meteo AQ API)  
вЂў **DuckDB** for analytical SQL over the cleaned hourly table  
вЂў **XGBoost** regression for 24 h ahead PM2.5 with a documented train/test split  
вЂў **Policy layer**: threshold CSV + indicative exceedance summary for the UI  
вЂў Reproducible **`urban_aqi`** package + `scripts/run_pipeline.py` + aligned notebooks

2. Problem statement

Urban pollution varies with traffic, weather, and season. The project addresses:

вЂў Explaining **recent** hourly conditions per city in a common format  
вЂў Providing a **model validation** view (seven-day window on the 2025 hold-out year)  
вЂў Comparing **mean levels** across cities with clear caveats on mixed sources  
вЂў Supporting **policy messaging** with banded thresholds and cautious language

3. Data sources

3.1 EEA (Berlin, Madrid)

Hourly parquet exports under `data/raw/eea/unzipped/`. Stations are mapped to cities using **Germany** (`DE/` / DEBEвЂ¦) for Berlin and **Madrid community** (`ES/` + `SP_28`вЂ¦) for Madrid. Pollutants used: PM2.5, PM10, NO2, O3 where present.

3.2 Open-Meteo weather (all four cities)

Historical weather CSVs (`data/raw/weather/openmeteo_<city>_2022-01-01_2025-12-31.csv`) supply temperature, humidity, pressure, precipitation, and wind for merge keys.

3.3 Open-Meteo air quality (Paris, Rome)

The bundled EEA extract does not include French or Italian stations. Paris and Rome use the **Open-Meteo Air Quality API** (CAMS European domain) at fixed coordinates, merged on time with weather. This is documented in the README and in-app captions so reviewers can assess **comparability** with EEA in-situ series.

3.4 Policy thresholds

`data/raw/rules/eu_aqi_policy_thresholds.csv` defines band cut-offs used for categorical colouring and messaging.
